<h1>Python Developer&#8217;s Guide<a class="headerlink" href="#python-developer-s-guide" title="Permalink to this headline">¶</a></h1>
<p>This guide is a comprehensive resource for <a class="reference internal" href="#contributing"><span>contributing</span></a>
to <a class="reference external" href="https://www.python.org/">Python</a> &#8211; for both new and experienced contributors.  It is
<a class="reference internal" href="docquality.html#helping-with-the-developers-guide"><span>maintained</span></a> by the same community
that maintains Python.  We welcome your contributions to Python!</p>
<div class="section" id="quick-start">
<h2>Quick Start<a class="headerlink" href="#quick-start" title="Permalink to this headline">¶</a></h2>
<p>Here are the basic steps needed to get <a class="reference internal" href="setup.html#setup"><span>set up</span></a> and contribute a
patch:</p>
<ol class="arabic">
<li><p class="first">Install and set up <a class="reference internal" href="setup.html#vcsetup"><span>Git</span></a> and other dependencies
(see the <a class="reference internal" href="setup.html#setup"><span>Get Setup</span></a> page for detailed information).</p>
</li>
<li><p class="first">Fork <a class="reference external" href="https://github.com/python/cpython">the CPython repository</a>
to your GitHub account and <a class="reference internal" href="setup.html#checkout"><span>get the source code</span></a> using:</p>
<div class="highlight-python"><div class="highlight"><pre><span></span>git clone https://github.com/&lt;your_username&gt;/cpython
</pre></div>
</div>
</li>
<li><p class="first">Build Python, on UNIX and Mac OS use:</p>
<div class="highlight-python"><div class="highlight"><pre><span></span>./configure --with-pydebug &amp;&amp; make -j
</pre></div>
</div>
<p>and on Windows use:</p>
<div class="highlight-python"><div class="highlight"><pre><span></span>PCbuild\build.bat -e -d
</pre></div>
</div>
<p>See also <a class="reference internal" href="setup.html#compiling"><span>more detailed instructions</span></a>,
<a class="reference internal" href="setup.html#build-dependencies"><span>how to build dependencies</span></a>, and the
plaform-specific pages for <a class="reference internal" href="setup.html#unix-compiling"><span>UNIX</span></a>,
<a class="reference internal" href="setup.html#macos"><span>Mac OS</span></a>, and <a class="reference internal" href="setup.html#windows-compiling"><span>Windows</span></a>.</p>
</li>
<li><p class="first"><a class="reference internal" href="runtests.html"><em>Run the tests</em></a>:</p>
<div class="highlight-python"><div class="highlight"><pre><span></span>./python -m test -j3
</pre></div>
</div>
<p>On <a class="reference internal" href="setup.html#mac-python-exe"><span>most</span></a> Mac OS X systems, replace <code class="file docutils literal"><span class="pre">./python</span></code>
with <code class="file docutils literal"><span class="pre">./python.exe</span></code>.  On Windows, use <code class="file docutils literal"><span class="pre">python.bat</span></code>.  With Python
2.7, replace <code class="docutils literal"><span class="pre">test</span></code> with <code class="docutils literal"><span class="pre">test.regrtest</span></code>.</p>
</li>
<li><p class="first">Create a new branch where your work for the issue will go, e.g.:</p>
<div class="highlight-python"><div class="highlight"><pre><span></span>git checkout -b fix-issue-12345 master
</pre></div>
</div>
<p>If an issue does not already exist, please <a class="reference external" href="https://bugs.python.org/">create it</a>.  Trivial issues (e.g. typo fixes) do not
require any issue to be created.</p>
</li>
<li><p class="first">Once you fixed the issue, run the tests, run <code class="docutils literal"><span class="pre">make</span> <span class="pre">patchcheck</span></code>, and if
everything is ok, commit.</p>
</li>
<li><p class="first">Push the branch on your fork on GitHub and <a class="reference internal" href="pullrequest.html"><em>create a pull request</em></a>.  Include the issue number using <code class="docutils literal"><span class="pre">bpo-NNNN</span></code> in the
pull request description.  For example:</p>
<div class="highlight-python"><div class="highlight"><pre><span></span>bpo-12345: Fix some bug in spam module
</pre></div>
</div>
</li>
</ol>
<div class="admonition note">
<p class="first admonition-title">Note</p>
<p class="last">First time contributors will need to sign the Contributor Licensing
Agreement (CLA) as described in the <a class="reference internal" href="pullrequest.html#cla"><span>Licensing</span></a> section of
this guide.</p>
</div>
</div>

<div class="section" id="proposing-changes-to-python-itself">
<h2>Proposing changes to Python itself<a class="headerlink" href="#proposing-changes-to-python-itself" title="Permalink to this headline">¶</a></h2>
<p>Improving Python&#8217;s code, documentation and tests are ongoing tasks that are
never going to be &#8220;finished&#8221;, as Python operates as part of an ever-evolving
system of technology.  An even more challenging ongoing task than these
necessary maintenance activities is finding ways to make Python, in the form of
the standard library and the language definition, an even better tool in a
developer&#8217;s toolkit.</p>
<p>While these kinds of change are much rarer than those described above, they do
happen and that process is also described as part of this guide:</p>
<ul class="simple">
<li><a class="reference internal" href="stdlibchanges.html"><em>Adding to the Stdlib</em></a></li>
<li><a class="reference internal" href="langchanges.html"><em>Changing the Python Language</em></a></li>
</ul>
</div>
<div class="section" id="other-interpreter-implementations">
<h2>Other Interpreter Implementations<a class="headerlink" href="#other-interpreter-implementations" title="Permalink to this headline">¶</a></h2>
<p>This guide is specifically for contributing to the Python reference interpreter,
also known as CPython (while most of the standard library is written in Python,
the interpreter core is written in C and integrates most easily with the C and
C++ ecosystems).</p>
<p>There are other Python implementations, each with a different focus.  Like
CPython, they always have more things they would like to do than they have
developers to work on them.  Some major example that may be of interest are:</p>
<ul class="simple">
<li><a class="reference external" href="http://www.pypy.org/">PyPy</a>: A Python interpreter focused on high speed (JIT-compiled) operation
on major platforms</li>
<li><a class="reference external" href="http://www.jython.org/">Jython</a>: A Python interpreter focused on good integration with the Java
Virtual Machine (JVM) environment</li>
<li><a class="reference external" href="http://ironpython.net/">IronPython</a>: A Python interpreter focused on good integration with the
Common Language Runtime (CLR) provided by .NET and Mono</li>
<li><a class="reference external" href="http://www.stackless.com/">Stackless</a>: A Python interpreter focused on providing lightweight
microthreads while remaining largely compatible with CPython specific
extension modules</li>
</ul>
</div>

<div class="section" id="code-of-conduct">
<h2>Code of Conduct<a class="headerlink" href="#code-of-conduct" title="Permalink to this headline">¶</a></h2>
<p>Please note that all interactions on
<a class="reference external" href="https://www.python.org/psf-landing/">Python Software Foundation</a>-supported infrastructure is
<a class="reference external" href="https://www.python.org/psf/records/board/minutes/2014-01-06/#management-of-the-psfs-web-properties">covered</a>
by the <a class="reference external" href="https://www.python.org/psf/codeofconduct/">PSF Code of Conduct</a>, which includes all
infrastructure used in the development of Python itself (e.g. mailing lists, issue trackers, GitHub, etc.).
In general this means everyone is expected to be open, considerate, and respectful of others no matter what
their position is within the project.</p>
